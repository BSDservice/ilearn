РАБОТА С УДАЛЁННЫМИ РЕПОЗИТОРИЯМИ


"origin" — имя по умолчанию для исходного репозитория


Просмотр удаленного репозитория

	git remote - выводит названия доступных удалённых репозиториев.
		-v - просмотреть адреса для чтения и записи, привязанные к репозиторию.

	git remote add [shortname] [url] - добавить удалённый репозиторий и присвоить ему имя (shortname)

	git remote show [remote-name] - выдаёт URL удалённого репозитория, а также информацию об отслеживаемых ветках.
		Данная команда показывает какая именно локальная ветка будет отправлена на удалённый сервер по умолчанию при выполнении git push. Она также
		показывает, каких веток с удалённого сервера у вас ещё нет, какие ветки всё ещё есть у вас, но уже удалены на сервере. И для нескольких веток
		показано, какие удалённые ветки будут в них влиты при выполнении git pull.

	
Получение изменений из удалённого репозитория

	git fetch [remote-name] - команда связывается с указанным удалённым проектом и забирает все те данные проекта, которых у вас ещё нет. После того
	 	как вы выполнили команду, у вас должны появиться ссылки на все ветки из этого удалённого проекта. Теперь эти ветки в любой момент могут быть
	 	просмотрены или слиты.

	git pull - автоматически получить изменения из удалённой ветви и слить их со своей текущей ветвью, если у вас есть ветка, настроенная на
	 	отслеживание удалённой ветки. Выполнение git pull, как правило, извлекает (fetch) данные с сервера, с которого вы изначально склонировали, и
	 	автоматически пытается слить (merge) их с кодом, над которым вы в данный момент работаете.


Отправка изменений в удаленный репозиторий 

	git push [remote-name] [branch-name] - отправит ветку branch-name на сервер remote-name.
		Эта команда срабатывает только в случае, если вы клонировали с сервера, на котором у вас есть права на запись, и если никто другой с тех пор не
		выполнял команду push. Если вы и кто-то ещё одновременно клонируете, затем он выполняет команду push, а затем команду push выполняете вы, то
		ваш push точно будет отклонён. Вам придётся сначала вытянуть (pull) их изменения и объединить с вашими. Только после этого вам будет позволено
		выполнить push.


Удаление и переименование удалённых репозиториев
	
	git remote rename [short-name] [new-short-name] - изменит сокращённое имя, используемое для удалённого репозитория

	git remote rm [short-name] - удаляет ссылку
